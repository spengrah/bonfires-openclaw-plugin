# Wave 5 Plan (20260302w5a)

## Scope (in)
1. Implement PM3 hosted integration verification workflow as a runnable script (`verify:hosted`).
2. Include mandatory preflight probes (`GET /healthz`, auth+bonfire access via `POST /generate_summaries`) with explicit live mode.
3. Include contract probes for `/delve`, `/stack/add`, and `/stack/process` using deterministic fixture mode.
4. Emit verification report artifact under `.ai/log/plan/` with timestamp + per-probe pass/fail and no secret leakage.
5. Refresh README to document hosted verification command and current post-MVP capabilities.

## Non-goals (out)
1. No changes to production runtime behavior of retrieval/capture/heartbeat flows.
2. No rollout of PM4 ingestion cron/hash ledger.
3. No mandatory inclusion of hosted live probes in `gate:all` (kept opt-in due environment dependency).

## Constraints
1. Verification output must redact secrets (only presence metadata, never key values).
2. Fixture-mode verification must run offline/deterministically.
3. Existing deterministic gates remain green (`npm run gate:all`).

## Acceptance criteria
1. `npm run verify:hosted` succeeds in fixture mode and writes `.ai/log/plan/hosted-integration-verification-current.json`.
2. Report includes probes for healthz, generate_summaries, delve normalization, stack/add mapping, stack/process response handling.
3. Report is secret-safe (no bearer/API key values).
4. README includes Wave 5 verification command + brief feature status update.

## Pass/fail gates
1. `npm run lint` => PASS
2. `npm run test` => PASS
3. `npm run verify:hosted` => PASS (fixture mode)
4. `npm run gate:all` => PASS

## Open blockers
- Live preflight probes depend on external hosted API availability and credentials.

## Assumption checks
1. **Baseline green:** PASS.
2. **Scope = risk scope:** yes (verification tooling + docs only).
3. **Gate coverage:** all acceptance criteria mapped to runnable commands.
4. **Spec/guidance alignment:** PM3 spec/guidance directly drives this wave scope.
